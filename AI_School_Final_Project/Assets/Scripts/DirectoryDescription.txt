Database 
  프로젝트 내 사용되는 데이터셋들이 정의될 폴더
  (Bo, Dto 데이터가 정의됨)
   Bo -> 인게임 내에서만 사용되는 데이터 
   (서버에서 받은 dto 데이터를 bo로 가공하여 사용, 
    또는 서버와의 통신 없이 생성할 수 있는 데이터라면 Sd(기획)데이터를 통해서 만들 수도 있음)
   Dto -> 서버와의 통신에 사용되는 데이터 
   (클라이언트가 서버에 데이터를 요청/반환 받을 시 dto의 형태로 전달됨)

Dummy
  프로젝트 내 사용될 더미 데이터 및 기능이 정의될 폴더
  (우리는 실제 서버를 사용하는 것이 아닌 더미서버를 사용한다.
   이 말은 클라이언트가 서버의 역할을 동시에 한다는 것)

Network
  통신 관련 기능 등이 정의될 폴더 
 (해당 폴더의 기능들을 이용하여 서버와의 데이터 통신을 한다.)

Object
  인게임 내 사용되는 객체들이 정의될 폴더
  (객체 정의 시에는 객체는 데이터와 기능 모듈(컨트롤러)를 갖는 식으로 작성되는 것이 베스트)

Controller
  게임 내 반복적으로 사용되는 기능이 존재한다면, 해당 기능을 수행할 수 있는 컨트롤러를 정의할 폴더
  결과적으로, 컨트롤러를 다양한 object들이 사용하게 됨
  (ex: 공격컨트롤러, 이동컨트롤러 등)

Resource
  프로젝트 내 리소스 관련 기능이 정의될 폴더
  (ex: 리소스를 부르는 기능, 특정 타입의 리소스를 제어하는 기능 등)

StaticData
  기획 데이터 기능 및 데이터셋이 정의될 폴더
  (기획 데이터는 일반적으로 고정된 데이터이기 때문에 저는 임의로 StaticData라고 표현)

UI
  UI 관련 기능이 정의될 폴더

Util
  그 외 프로젝트에 전반적으로 사용되는 기능 등이 정의될 폴더
